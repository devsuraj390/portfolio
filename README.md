# Suraj Kumar Portfolio

Minimal static portfolio site prepared for GitHub Pages.

## Local preview

Open `index.html` directly in the browser, or run a simple static server from this folder.

## Deploy to GitHub Pages

1. Create a GitHub repository, for example `portfolio`.
2. Push this folder to the repository.
3. In GitHub, open `Settings` -> `Pages`.
4. Under `Build and deployment`, choose `Deploy from a branch`.
5. Select the `main` branch and `/ (root)` folder.
6. Save. GitHub Pages will publish the site.

Recommended repository name: `portfolio`.

Between `portfolio` and `devsuraj390`, `portfolio` is the cleaner project repository name.

If you want the site to live at `https://devsuraj390.github.io/`, the repository must be named `devsuraj390.github.io`.
